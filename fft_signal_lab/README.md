## 1. Что реализовано

Реализован небольшой исследовательский C++-проект по цифровой обработке одномерных сигналов. Проект строится вокруг одной центральной идеи: дискретный сигнал можно анализировать не только во временной области, но и в частотной области через быстрое преобразование Фурье.

В программе есть три режима работы:

1. `detect` - распознавание основных частот в сигнале.
2. `reconstruct` - построение более плотной временной сетки по цифровым отсчетам через частотную интерполяцию.
3. `denoise` - подавление шума через отбор значимых спектральных компонент.

Также реализованы:

- чтение сигнала из CSV;
- запись результата в CSV;
- генератор синтетических сигналов;
- генератор белого гауссовского шума;
- оконные функции `rectangular`, `hann`, `hamming`, `blackman`;
- параболическое уточнение частоты спектрального пика;
- встроенный smoke-test для проверки корректности ключевых режимов.

## 3. Математическая основа

### 3.1. Дискретный сигнал

В программе сигнал хранится как последовательность вещественных отсчетов:

```text
x[0], x[1], ..., x[N - 1]
```

и частота дискретизации:

```text
fs
```

Отсчет `x[n]` соответствует времени:

```text
t_n = n / fs
```


### 3.2. Дискретное преобразование Фурье

Идея ДПФ: представить конечную последовательность как сумму комплексных гармоник. Для каждого частотного индекса `k` получается комплексное число `X[k]`, которое содержит информацию об амплитуде и фазе соответствующей гармоники.

Частота, соответствующая бину `k`, вычисляется так:

```text
f_k = k * fs / N
```

где:

- `fs` - частота дискретизации;
- `N` - длина FFT;
- `k` - номер спектрального бина.

Модуль `|X[k]|` показывает силу частотной компоненты. Фаза вычисляется как аргумент комплексного числа.

### 3.3. Соглашение о знаке в FFT

В проекте используется FFT-реализация, которая была у нас в курсе мат основ алгоритмов. В ней прямое преобразование использует положительный знак в экспоненте, а обратное - отрицательный:

```cpp
alpha = 2 * PI * i / n * (invert ? -1 : 1)
```

Это противоположно классической записи, где прямое ДПФ часто пишется с отрицательным знаком. Это не ломает проект, ведь модуль спектра не меняется, распознавание частот работает корректно, восстановление работает корректно, потому что прямое и обратное преобразования используются согласованной парой.



### 3.4. Зачем нужны окна

Если взять конечный кусок гармонического сигнала, то в частотной области получается не идеальная дельта на одной частоте, а пик с боковыми лепестками.

Причина: конечный фрагмент сигнала эквивалентен умножению бесконечного сигнала на прямоугольное окно. Умножение во временной области соответствует свертке в частотной области, поэтому энергия частоты размазывается по соседним бинам.

В проекте реализованы окна:

- `rectangular` - без сглаживания;
- `hann` - хороший базовый выбор для спектрального анализа;
- `hamming` - похожее окно с другим уровнем боковых лепестков;
- `blackman` - сильнее подавляет боковые лепестки, но шире главный лепесток.

Для режима `detect` по умолчанию используется `hann`.

### 3.5. Параболическое уточнение пика

FFT дает частоты только на сетке:

```text
0, fs/N, 2fs/N, 3fs/N, ...
```

Если настоящая частота лежит между двумя бинами, максимум спектра окажется около ближайшего бина, но не обязательно точно в нем. Поэтому после нахождения локального максимума используется параболическое уточнение по трем значениям:

```text
m[k - 1], m[k], m[k + 1]
```

Сдвиг вершины параболы:

```text
delta = 0.5 * (m[k - 1] - m[k + 1]) / (m[k - 1] - 2m[k] + m[k + 1])
```

После этого уточненная частота:

```text
f = (k + delta) * fs / N
```


## 4. Режим `detect`

### 4.1. Назначение

Режим `detect` ищет основные частоты в сигнале.

Он подходит для сигналов вида:

```text
x(t) = A1 sin(2π f1 t + φ1) + A2 sin(2π f2 t + φ2) + noise
```

То есть он хорошо работает, когда сигнал содержит несколько выраженных гармонических компонент.

### 4.2. Последовательность обработки

```text
samples
  -> удаление DC-составляющей
  -> применение оконной функции
  -> zero padding или обрезка до nfft
  -> FFT
  -> односторонний амплитудный спектр
  -> поиск локальных максимумов
  -> параболическое уточнение пиков
  -> вывод top-k частот
```

Удаление DC-составляющей нужно, чтобы постоянное смещение сигнала не доминировало в нулевом бине.

Окно нужно, чтобы уменьшить влияние резкого обрыва анализируемого фрагмента.

Односторонний спектр используется потому, что входной сигнал вещественный. Для вещественного сигнала спектр симметричен: положительные и отрицательные частоты несут дублирующую информацию.

### 4.3. Пример запуска

```bash
./dsp_lab detect --generate "sine:440:1.0,sine:880:0.4,noise:0.02" --fs 8192 --duration 1 --nfft 8192 --window hann --top 5
```

Примерный вывод:

```text
rank,frequency_hz,magnitude,phase_rad
1,440.000000,...,...
2,880.000000,...,...
```

### 4.4. Аргументы режима

```text
--input <path>       CSV-файл с сигналом
--generate <spec>    синтетический сигнал
--fs <value>         частота дискретизации
--duration <value>   длительность синтетического сигнала
--nfft <value>       длина FFT
--window <type>      rectangular, hann, hamming, blackman
--top <value>        сколько пиков вывести
--floor <value>      минимальный относительный уровень пика
```

## 5. Режим `reconstruct`

### 5.1. Назначение

Режим `reconstruct` строит более плотную временную сетку по исходным цифровым отсчётам. На вход подаётся дискретный сигнал с частотой дискретизации `fs`, а на выходе получается новая последовательность отсчётов с частотой дискретизации

\[
fs_{out} = factor \cdot fs.
\]

Важно понимать, что программа не восстанавливает произвольный аналоговый сигнал. Режим корректно работает только в модели, где исходный непрерывный сигнал был достаточно хорошо дискретизирован.

словие задаётся теоремой Котельникова. Если аналоговый сигнал не содержит частот выше некоторой границы \( f_{max} \), то для его однозначного восстановления по отсчётам необходимо, чтобы частота дискретизации была больше удвоенной максимальной частоты:

\[
fs > 2 f_{max}.
\]


### 5.2. Последовательность обработки

```text
samples длины N
  -> дополнение до ближайшей степени двойки
  -> FFT
  -> зануление частот выше cutoff
  -> перенос низких положительных и отрицательных частот в спектр большего размера
  -> inverse FFT
  -> масштабирование амплитуды на factor
  -> запись результата в CSV
```

Если исходный сигнал имеет `N` отсчетов, а `factor = L`, то после восстановления получается примерно `L * N_fft` отсчетов, где `N_fft` - ближайшая степень двойки не меньше исходной длины.

Новая частота дискретизации:

```text
fs_out = factor * fs_in
```

### 5.3. Почему появляется множитель `factor`

При обратном FFT результат делится на длину обратного преобразования. Если мы расширяем спектр с длины `N` до длины `M = factor * N`, то обратное преобразование нормирует результат уже на `M`, а не на `N`.

Чтобы амплитуда не уменьшилась в `factor` раз, после inverse FFT результат умножается на `factor`.

### 5.4. Пример запуска

```bash
./dsp_lab reconstruct --generate "sine:32:1.0,sine:64:0.5" --fs 1024 --duration 1 --factor 4 --cutoff 100 --output reconstructed.csv
```

Файл `reconstructed.csv` будет иметь вид:

```csv
time,value
0.000000000000,0.000000000000
0.000244140625,...
...
```


### 5.5. Ограничения режима

Режим работает хорошо, если:

- сигнал достаточно хорошо дискретизирован;
- существенные частоты ниже `cutoff`;
- отсутсвует эффект наложения частот при недостаточно высокой частоте дискретизации;
- сигнал не имеет резких нестационарных скачков, которые требуют большого числа высоких частот.

Режим работает плохо, если:

- частота дискретизации была слишком маленькой;
- полезный сигнал широкополосный;
- cutoff выбран слишком маленьким или слишком большим;
- на входе очень короткий фрагмент.

## 6. Режим `denoise`

### 6.1. Назначение

Режим `denoise` подавляет шум через спектральную маску.

Идея простая: если полезный сигнал состоит из нескольких выраженных гармоник, то в спектре они дают большие пики. Белый шум распределен по частотам более равномерно. Значит, можно оставить сильные частотные компоненты, а слабые занулить.

### 6.2. Последовательность обработки

```text
noisy samples
  -> FFT
  -> вычисление модулей спектральных компонент
  -> выбор значимых частот
  -> зануление остальных частот
  -> inverse FFT
  -> запись очищенного сигнала
```

Есть два способа выбора частот:

1. `--threshold` - оставить все компоненты выше относительного порога.
2. `--keep-top` - оставить только `k` самых сильных частотных пар.

Для вещественного сигнала важно сохранять комплексно-сопряженные пары частот. Если оставить только положительную частоту и забыть отрицательную, после обратного преобразования можно получить комплексный или искаженный сигнал. В реализации для каждого бина `k` сохраняется и бин `N-k`.

### 6.3. Пример запуска через threshold

```bash
./dsp_lab denoise --generate "sine:440:1.0,sine:880:0.5,noise:0.4" --fs 8192 --duration 1 --threshold 0.05 --output clean.csv
```

### 6.4. Пример запуска через keep-top

```bash
./dsp_lab denoise --generate "sine:440:1.0,sine:880:0.5,noise:0.4" --fs 8192 --duration 1 --keep-top 2 --output clean.csv
```

Для синтетического сигнала из двух синусоид `--keep-top 2` обычно работает очень хорошо.

### 6.5. Ограничения режима

Этот режим не является универсальным шумодавом. Он хорошо работает, если полезный сигнал сосредоточен в малом числе частотных компонент. Он будет плохим решением для речи, музыки, ударных сигналов и широкополосных полезных процессов, потому что там полезная информация сама занимает широкий частотный диапазон.

## 7. Формат CSV

Программа умеет читать два простых варианта CSV.

Вариант 1: один столбец значений:

```csv
0.000000
0.338738
0.637424
0.860742
```

Вариант 2: два столбца `time,value`:

```csv
time,value
0.000000,0.000000
0.000125,0.338738
0.000250,0.637424
```

Если в строке есть два числа, программа берет второй столбец как значение сигнала. Частоту дискретизации все равно нужно передавать через `--fs`, потому что CSV может не иметь надежного временного столбца.

## 8. Генератор сигналов

Поддерживаются компоненты:

```text

cos:freq:amplitude[:phase]
noise:sigma

```

Примеры:

```bash
--generate "sine:440:1.0"
```

```bash
--generate "sine:440:1.0,sine:880:0.4,noise:0.05"
```

```bash
--generate "dc:1.0,sine:100:0.5"
```

## 11. Как проверять вручную

### 11.1. Проверка распознавания частот

```bash
./dsp_lab detect --generate "sine:440:1.0,sine:880:0.4,noise:0.02" --fs 8192 --duration 1 --nfft 8192 --window hann --top 5
```

Нужно увидеть пики около 440 Hz и 880 Hz.

### 11.2. Проверка восстановления

```bash
./dsp_lab reconstruct --generate "sine:32:1.0,sine:64:0.5" --fs 1024 --duration 1 --factor 4 --cutoff 100 --output reconstructed.csv
```

После этого можно построить график. Кривая должна быть гладкой.

### 11.3. Проверка шумоподавления

```bash
./dsp_lab denoise --generate "sine:440:1.0,sine:880:0.5,noise:0.4" --fs 8192 --duration 1 --keep-top 2 --output clean.csv
```

Затем можно отдельно сгенерировать чистый сигнал и сравнить графики.

## 12. Описание файлов

### 12.1. `signal.h` / `signal.cpp`

Здесь находится структура `Signal`:

```cpp
struct Signal {
    double sample_rate;
    std::vector<double> samples;
};
```

Также реализованы:

- чтение CSV;
- запись CSV;
- вычисление среднего;
- удаление среднего;
- MSE между двумя последовательностями.

`remove_mean` используется в спектральном анализе, чтобы убрать постоянную составляющую.

### 12.2. `fft.h` / `fft.cpp`

Здесь находится реализация FFT.

Также есть вспомогательные функции:

- `next_power_of_two`;
- `to_complex`;
- `real_part`.

FFT требует, чтобы длина массива была степенью двойки. Поэтому в режимах обработки размер приводится к ближайшей степени двойки.

### 12.3. `window.h` / `window.cpp`

Здесь реализованы оконные функции и их применение к сигналу.

Функция `make_window(type, n)` возвращает массив коэффициентов окна длины `n`.

Функция `apply_window` поэлементно умножает сигнал на окно.

### 12.4. `spectrum.h` / `spectrum.cpp`

Здесь реализованы:

- построение амплитудного спектра;
- построение спектра мощности;
- вычисление фаз;
- поиск локальных максимумов;
- параболическое уточнение частоты;
- печать найденных пиков.

Главная функция:

```cpp
SpectrumAnalysis analyze_spectrum(
    const Signal& signal,
    int nfft,
    const std::string& window_type,
    bool subtract_mean
);
```

### 12.5. `reconstruction.h` / `reconstruction.cpp`

Здесь находятся два алгоритма:

```cpp
Signal reconstruct_by_fft_upsampling(
    const Signal& input,
    int factor,
    double cutoff_hz
);
```

и

```cpp
Signal spectral_denoise(
    const Signal& input,
    int nfft,
    double threshold,
    int keep_top
);
```

Первый алгоритм делает частотную интерполяцию. Второй алгоритм делает спектральную маску и обратное преобразование.

### 12.6. `generator.h` / `generator.cpp`

Здесь реализован генератор синтетических сигналов. Он нужен, чтобы проект можно было проверить без внешних файлов.

### 12.7. `cli.h` / `cli.cpp`

Здесь находится разбор аргументов командной строки, маршрутизация режимов и smoke-test.
